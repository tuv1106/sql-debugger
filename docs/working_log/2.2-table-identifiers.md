# Task 2.2: Table Identifiers

## Goal
Represent fully-qualified table names across DB types with a consistent interface for exclusion checks, sqlglot integration, and API references.

## Files to Create
1. `local_backend/models/tables.py` — TableIdentifier base + 3 subclasses
2. `tests/local_backend/test_models_tables.py` — 7 tests

## Models to Implement

### `local_backend/models/tables.py`
- `TableIdentifier(BaseModel)` — base class
  - Abstract methods: `full_name() -> str`
  - Concrete methods: `to_dict() -> dict`, `from_dict(data: dict) -> TableIdentifier` (class method, dispatches by `db_type`)
- `PostgresTable(TableIdentifier)` — fields: `schema_name`, `table_name` → full_name: `"schema.table"`
- `MySQLTable(TableIdentifier)` — fields: `database`, `table_name` → full_name: `"database.table"`
- `BigQueryTable(TableIdentifier)` — fields: `project`, `dataset`, `table_name` → full_name: `"project.dataset.table"`

### Design Decisions
- Use Pydantic BaseModel for consistency with other models
- `from_dict` is a factory on the base class that reads `db_type` to pick the right subclass
- Field names match the DB terminology (schema_name not schema, to avoid Python keyword collision)

## Tests (7)

1. **Postgres full_name** — `PostgresTable(schema_name="public", table_name="orders").full_name()` == `"public.orders"`
2. **MySQL full_name** — `MySQLTable(database="mydb", table_name="orders").full_name()` == `"mydb.orders"`
3. **BigQuery full_name** — `BigQueryTable(project="proj", dataset="ds", table_name="orders").full_name()` == `"proj.ds.orders"`
4. **Postgres serialization roundtrip** — `to_dict()` then `from_dict()` returns equal object
5. **MySQL serialization roundtrip** — same
6. **BigQuery serialization roundtrip** — same
7. **from_dict with invalid db_type** — raises ValueError
